# MapBook PC 端交互设计文档

## 1. 设计原则

- **地图优先**：地图是核心操作区，占据主要视口
- **渐进式交互**：复杂操作分步引导，降低认知负担
- **即时反馈**：地图操作实时响应，状态变化有明确视觉提示
- **键盘友好**：支持常用快捷键，提升编辑效率

---

## 2. 页面结构

### 2.1 整体布局

```
┌──────────────────────────────────────────────────┐
│  Header (Logo | 导航 | 用户头像)                    │
├──────────────────────────────────────────────────┤
│         │                                │        │
│  Side-  │                                │        │
│  bar    │         Map Canvas              │        │
│  行程    │         (高德地图)               │        │
│  面板    │                                │        │
│         │                                │        │
│ [Day1]  │    📍 标记点                    │        │
│ [Day2]  │    ── 路线连线                  │        │
│ [Day3]  │    🔵 搜索框                    │        │
│         │                                │        │
│ +添加天  │                                │        │
│         │                                │        │
├─────────┴─────────────────────────────────────────┤
│  底部状态栏 (距离 | 时长 | 当前操作提示)               │
└──────────────────────────────────────────────────┘
```

### 2.2 路由与页面

| 路由 | 页面 | 描述 |
|------|------|------|
| `/` | 首页 | 全屏地图 + 搜索框，快速创建路书入口 |
| `/trips` | 路书列表 | 卡片网格展示所有路书 |
| `/trips/new` | 新建路书 | 地图编辑器模式，左侧行程面板空 |
| `/trips/:id` | 编辑路书 | 地图编辑器模式，加载已有数据 |
| `/auth` | 登录/注册 | 邮箱登录表单 |

## 3. 核心交互流程

### 3.1 创建路书流程

```
首页 → 点击"创建路书" → 弹出信息表单（标题、描述）
     → 确认 → 跳转 /trips/new 编辑页
     → 地图上搜索/点击添加地点
     → 左侧面板出现 Day 1
     → 继续添加地点、生成路线
     → 自动保存（防抖 2s）
     → 点击"完成"返回列表
```

### 3.2 添加地点交互

| 方式 | 操作 | 反馈 |
|------|------|------|
| 搜索添加 | 在搜索框输入关键词 → 下拉出现候选列表 → 点击选中 | 地图定位到该点，自动添加标记 |
| 地图点击 | 在地图上右键/长按某位置 | 弹出菜单"设为起点/终点/途径点" |
| 拖拽标记 | 按住已有标记拖动 | 路线实时重绘 |

### 3.3 按天规划交互

```
左侧行程面板：
┌──────────────────────┐
│ 🗓 Day 1 (6月25日)   │  ← 可折叠
│  ├ 📍 北京·天安门    │  ← 可拖拽排序
│  ├ 📍 北京·故宫      │  ← 点击选中，地图高亮
│  ├ 📍 北京·颐和园    │  ← 悬停显示删除按钮
│  └ ➕ 添加地点       │
│ ── 路线: 15km · 40min│  ← 当前天统计
├──────────────────────┤
│ 🗓 Day 2 (6月26日)   │
│  ├ 📍 天津·瓷房子    │
│  └ ➕ 添加地点       │
├──────────────────────┤
│ ➕ 添加新的一天       │
└──────────────────────┘
```

### 3.4 路线生成交互

```
1. 用户在 Day 1 添加至少 2 个地点
2. 系统自动调用高德路径规划 API
3. 地图上绘制彩色路线（每 Day 不同颜色）
4. 路线旁显示距离和预计时长
5. 如添加/删除/移动地点，路线自动更新
```

## 4. 状态与反馈

### 4.1 Loading 状态
| 场景 | 处理 |
|------|------|
| 地图 SDK 加载 | 全屏骨架屏 + 加载进度条 |
| 路线计算中 | 路线区域显示虚线动画 |
| 数据保存 | 左下角 Toast "已保存" |
| 搜索结果 | 输入框内 loading 图标 |

### 4.2 空状态
| 场景 | 处理 |
|------|------|
| 无路书 | 列表页显示插图 + "创建第一条路书"按钮 |
| 新建路书无地点 | 地图中央引导文字"搜索或点击地图添加地点" |
| 某天无地点 | 面板显示"点击地图添加当天行程" |

### 4.3 错误状态
| 场景 | 处理 |
|------|------|
| 路径规划失败 | Toast 提示"无法规划此路线，请调整地点" |
| 网络断连 | 顶部横幅"网络连接已断开" |
| 登录过期 | 弹窗引导重新登录 |

## 5. 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + S` | 手动保存 |
| `Delete` | 删除选中的地点 |
| `Esc` | 关闭弹窗/取消选中 |
| `1-9` | 快速切换到第 N 天 |
| `+/=` | 添加新的一天 |
| `Ctrl + Z` | 撤销（V2） |

## 6. PC 端视觉规范

### 6.1 布局尺寸
- 最小视口宽度：1024px
- 侧边栏宽度：360px（可拖拽调整 280-480px）
- 头部高度：56px
- 底部状态栏高度：32px

### 6.2 色彩系统
```
主题色：#1677FF (高德蓝)
成功：#52C41A
警告：#FAAD14
危险：#FF4D4F
文字：#1F1F1F / #666666 / #999999
背景：#FFFFFF / #F5F5F5
边框：#E8E8E8
```

### 6.3 路线颜色
| Day | 颜色 |
|-----|------|
| Day 1 | #1677FF (蓝) |
| Day 2 | #52C41A (绿) |
| Day 3 | #FAAD14 (橙) |
| Day 4 | #FF4D4F (红) |
| Day 5 | #722ED1 (紫) |
| Day 6+ | #13C2C2 (青) ... 循环 |

## 7. 响应式策略（V2 移动端）

```
PC (>1024px)：
  地图 + 侧边栏 并排

平板 (768-1024px)：
  侧边栏宽度缩减到 280px

移动端 (<768px)：
  地图全屏 + 底部抽屉面板
  上滑展开行程列表
  Google Maps 风格交互
```

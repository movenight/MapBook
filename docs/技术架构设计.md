# MapBook 技术架构设计

## 1. 技术选型

| 类别 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| 语言 | TypeScript | 5.x | 类型安全，提高代码质量 |
| 框架 | Vue 3 (Composition API) | 3.5+ | 渐进式、轻量、易上手，可通过 Uni-app 移植小程序 |
| 构建工具 | Vite | 5.x | 开发体验极快，HMR 热更新 |
| 状态管理 | Pinia | 2.x | Vue 3 官方推荐，轻量（<1KB） |
| 路由 | Vue Router | 4.x | SPA 路由，支持懒加载 |
| UI 库 | Element Plus | 2.x | 按需引入，组件齐全，中文友好 |
| 地图 | 高德地图 JS API 2.0 | — | 国内路由数据最精确 |
| 后端 | Supabase | — | 自带 Auth + PostgreSQL + Storage + RLS |
| HTTP | Supabase JS Client | 2.x | 内置 REST + Realtime，无需 Axios |
| CSS | Less | — | 嵌套语法，配合组件 scoped 样式 |
| 图标 | @element-plus/icons-vue | — | 与 Element Plus 配套 |
| 日历 | Day.js | 1.x | 2KB 轻量日期处理 |
| 代码质量 | ESLint + Prettier | — | 代码规范 + 自动格式化 |

## 3. 数据库设计 (Supabase PostgreSQL)

### 3.1 表结构

#### trips（路书表）
```sql
CREATE TABLE trips (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  title         TEXT NOT NULL DEFAULT '',
  description   TEXT DEFAULT '',
  cover_lng     DOUBLE PRECISION,        -- 封面地图中心经度
  cover_lat     DOUBLE PRECISION,        -- 封面地图中心纬度
  cover_zoom    INTEGER DEFAULT 10,
  start_date    DATE,
  end_date      DATE,
  day_count     INTEGER DEFAULT 1,
  status        TEXT DEFAULT 'draft' CHECK (status IN ('draft', 'published')),
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW()
);

-- RLS: 用户只能读写自己的路书
ALTER TABLE trips ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users manage own trips" ON trips
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```

#### waypoints（地点表）
```sql
CREATE TABLE waypoints (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  trip_id       UUID NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
  day_index     INTEGER NOT NULL DEFAULT 1,
  order_index   INTEGER NOT NULL DEFAULT 0,
  name          TEXT NOT NULL DEFAULT '',
  address       TEXT DEFAULT '',
  lng           DOUBLE PRECISION NOT NULL,
  lat           DOUBLE PRECISION NOT NULL,
  poi_id        TEXT DEFAULT '',          -- 高德 POI ID
  type          TEXT DEFAULT 'waypoint' CHECK (type IN (
                  'departure', 'destination', 'waypoint', 'lodgment', 'dining'
                )),
  notes         TEXT DEFAULT '',
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_waypoints_trip ON waypoints(trip_id, day_index, order_index);

ALTER TABLE waypoints ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users manage waypoints of own trips" ON waypoints
  USING (EXISTS (
    SELECT 1 FROM trips WHERE trips.id = waypoints.trip_id AND trips.user_id = auth.uid()
  ))
  WITH CHECK (EXISTS (
    SELECT 1 FROM trips WHERE trips.id = waypoints.trip_id AND trips.user_id = auth.uid()
  ));
```

#### routes（路线缓存表）
```sql
CREATE TABLE routes (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  trip_id       UUID NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
  day_index     INTEGER NOT NULL,
  polyline      TEXT NOT NULL,            -- JSON: [[lng,lat],...]
  distance      INTEGER DEFAULT 0,        -- 米
  duration      INTEGER DEFAULT 0,        -- 秒
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE routes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users manage routes of own trips" ON routes
  USING (EXISTS (
    SELECT 1 FROM trips WHERE trips.id = routes.trip_id AND trips.user_id = auth.uid()
  ));
```

## 5. 数据流设计

### 5.1 核心数据流
```
用户操作 → Composable → Pinia Store → Supabase API
                        ↕
                    Vue 组件响应式更新
```

### 5.2 路书编辑器数据流（关键）
```
┌────────────────────────────────────────────────────────┐
│  tripStore (Pinia)                                      │
│  ┌─────────────────────────────────────────────────────┐│
│  │ currentTrip: {                                     ││
│  │   id, title, description, startDate, endDate,      ││
│  │   waypoints: [                                     ││
│  │     { id, dayIndex, orderIndex, name, lng, lat,    ││
│  │       type, notes }                                ││
│  │   ],                                               ││
│  │   routes: Map<string, RouteData>  // key=dayIndex  ││
│  │ }                                                  ││
│  └─────────────────────────────────────────────────────┘│
│                                                         │
│  操作 Action:                                           │
│  ├── addWaypoint(dayIndex, waypoint)                   │
│  ├── removeWaypoint(id)                                │
│  ├── moveWaypoint(id, newDayIndex, newOrderIndex)      │
│  ├── updateWaypoint(id, patch)                         │
│  ├── addDay()                                          │
│  ├── removeDay(dayIndex)                               │
│  └── recalculateRoute(dayIndex)  → 调用高德API        │
│                                                         │
│  Getters:                                               │
│  ├── waypointsByDay(dayIndex)                          │
│  ├── totalDistance / totalDuration                     │
│  └── orderedWaypoints                                  │
└────────────────────────────────────────────────────────┘
```

### 5.3 自动保存机制
```
用户操作 → 修改 Store → 触发 debounce(2000ms)
                      → useTrip().save(tripStore.currentTrip)
                      → Supabase upsert
                      → 底部状态栏显示 "已保存 ✓"
```

## 6. 高德地图集成方案

### 6.1 SDK 加载策略
```typescript
// 使用 @amap/amap-jsapi-loader 按需异步加载
// services/amap.ts
import AMapLoader from '@amap/amap-jsapi-loader'

export async function loadAMap(): Promise<typeof AMap> {
  const AMap = await AMapLoader.load({
    key: import.meta.env.VITE_AMAP_KEY,    // 环境变量
    version: '2.0',
    plugins: [
      'AMap.Geocoder',       // 地理编码/逆地理编码
      'AMap.AutoComplete',   // 输入提示
      'AMap.PlaceSearch',    // POI 搜索
      'AMap.Driving',        // 驾车路径规划
      'AMap.Walking',        // 步行路径规划
      'AMap.Riding',         // 骑行路径规划
      'AMap.ContextMenu',    // 右键菜单
    ],
  })
  return AMap
}
```

### 6.2 地图组件封装（可替换性）
```
MapContainer.vue 是高德地图的唯一封装入口。
如需替换为其他地图 SDK（如 Leaflet），只需修改此组件。
```

### 6.3 API 调用流控
- POI 搜索：输入防抖 500ms
- 路径规划：防抖 2s，带缓存（同天同地点序列不重复请求）
- 地图标记：最多 200 个（超过手动静默）

## 8. 移动端 / 小程序扩展策略

### 8.1 Uni-app 迁移路径
```
Vue 3 + Composition API 代码 → 直接迁移到 Uni-app 的 Vue 3 模式

需替换的部分：
  ├── 地图组件：高德 JS API → uni-app 内置 <map> 组件（底层调高德原生 SDK）
  ├── 路由：Vue Router → uni-app pages.json
  ├── UI 库：Element Plus → Uni-ui / uView
  └── HTTP：Supabase JS Client 可保留（支持 H5 + 小程序）

可复用的部分（~70%）：
  ├── TypeScript 类型定义
  ├── Pinia Store 逻辑
  ├── Composable 业务逻辑（地图相关除外）
  ├── Supabase API 调用层
  └── 工具函数
```

### 8.2 组件设计原则
- 地图操作逻辑封装在 `composables/useMap.ts`，组件层只负责渲染
- 数据操作与 UI 分离，Store 层不引用任何 UI 库
- API 调用统一走 `services/api/`，方便切换后端
